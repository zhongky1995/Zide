3.2 架构分层原则

在目标架构里，最关键的是将“内容生产流程”与“模型调用细节”解耦。前者稳定、后者可替换，才能在后续模型升级时不动业务核心。

建议采用统一任务编排层，向上暴露固定操作语义（续写、扩写、重写等），向下按策略路由到不同模型。这样可以保证 UI 的可预测性，同时保留成本和质量优化空间。

为降低运维风险，所有跨章节 AI 操作必须落日志并可回滚。每次生成都记录输入摘要、上下文片段、模型参数和输出版本号，形成可追溯链路。
