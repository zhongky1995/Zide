# Zide V3 QA 验收脚本

## 测试前准备

1. 启动桌面端应用，创建一个 `小说` 类型项目。
2. 填写一个明确的小说 idea，确保系统可生成大纲与 Story Bible。
3. 进入 `Story Bible`，至少确认一次底座信息。
4. 进入 `Plot Board`，确保至少有 2 个章节目标，且为第 1 章填写 `objective / conflict / emotionalShift / payoff`。

---

## Q1 主流程验收

### 用例 1：从 Story Bible 走到候选稿

步骤：

1. 打开 `Story Bible`，点击 `确认 Story Bible`。
2. 打开 `Plot Board`，确认第 1 章目标已保存。
3. 打开 `Scene Sprint`，选中第 1 章。
4. 在 `任务补充说明` 中输入一句明确要求，例如“让主角第一次正面 столк上监察官，并显出迟疑到决断的情绪变化”。
5. 点击 `推进场景`。

通过标准：

1. 页面出现新的候选稿，而不是直接覆盖正文。
2. `Run Console` 能看到 route、Plan/Execute/Evaluate/Revise 步骤。
3. `Continuity Review` 能看到该候选稿对应的连续性报告。

不通过标准：

1. 点击后正文被直接改写；
2. 没有候选稿；
3. Run Console 没有步骤轨迹。

### 用例 2：通过后采纳进入正文

步骤：

1. 在 `Scene Sprint` 或 `Continuity Review` 里选中一份 `通过` 的候选稿。
2. 点击 `采纳到正文`。
3. 返回 `Scene Sprint` 或旧编辑器查看章节正文。
4. 打开 `Manuscript Center` 查看准备度。

通过标准：

1. 候选稿状态变成 `已采纳`；
2. 正文被更新；
3. `Manuscript Center` 可正常展示 readiness 和 blocker。

不通过标准：

1. 候选稿显示已采纳但正文没变；
2. 采纳后页面报错；
3. Manuscript Center 空白或无法刷新。

## Q2 异常流程验收

### 用例 3：连续性不通过时不污染正文

步骤：

1. 回到 `Scene Sprint`，针对同一章节发起一个明显偏离章节目标的任务。
2. 等待候选稿生成。
3. 打开 `Continuity Review`，确认该稿件为 `未通过`。
4. 不执行强制采纳，直接返回正文查看内容。

通过标准：

1. `Continuity Review` 明确显示未通过和问题列表；
2. 正文保持原样，没有被失败稿污染。

不通过标准：

1. 虽然报告未通过，但正文已被改写；
2. 无法定位是哪条问题导致未通过。

### 用例 4：强制采纳会自动创建快照

步骤：

1. 在 `Continuity Review` 里选中一份 `未通过` 的候选稿。
2. 点击 `强制采纳`。
3. 观察提示信息中的快照 ID，或进入快照相关页面查看。

通过标准：

1. 系统允许强制采纳；
2. 提示中出现自动快照信息；
3. 正文更新后仍可通过快照机制回退。

不通过标准：

1. 强制采纳成功但没有快照；
2. 强制采纳后正文更新异常。

### 用例 5：Retcon 后可定位受影响章节

步骤：

1. 打开 `Retcon Flow`。
2. 填写一个 retcon 摘要，至少选择 1 个章节，并可补受影响角色。
3. 点击 `创建提案`。
4. 在右侧列表点击 `批准并写入记忆`。
5. 对已批准 retcon 点击 `去复查`。

通过标准：

1. retcon 可创建并进入 `待批准`；
2. 批准后出现快照提示；
3. `Lore Memory` 中能看到对应 retcon 记忆；
4. 点击 `去复查` 后会跳到 `Continuity Review`，并定位到受影响章节。

不通过标准：

1. retcon 无法保存；
2. 批准后没有快照或没有长期记忆；
3. 去复查无法跳转。

---

## 结果记录建议

每个用例都记录三项：

1. 是否通过；
2. 如果失败，卡在第几步；
3. 页面看到的提示文案或截图。
